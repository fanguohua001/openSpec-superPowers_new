# AI 聊天实现计划

> **给执行代理的要求：** 实现本计划时必须使用 `superpowers:subagent-driven-development`（推荐）或 `superpowers:executing-plans`。任务使用复选框格式，便于逐项跟踪。

**目标：** 在 RuoYi-Vue 项目中接入可持久化、按用户隔离、支持流式响应的 AI 聊天功能。

**架构：** 后端沿用 RuoYi 分层：`ruoyi-admin` 放 Controller，`ruoyi-system` 放 domain、mapper、service 和模型适配器。前端沿用 Vue 2 + Element UI，在“系统工具”下新增 `tool/aiChat/index` 页面。模型服务配置从 `sys_config` 读取，默认适配 OpenAI Chat Completions 兼容 SSE。

**技术栈：** Java 8、Spring Boot 2.5.15、Spring MVC SSE、MyBatis XML、MySQL、Vue 2、Element UI、Axios、Maven、npm。

- [ ] **步骤 1：新增 AI 聊天会话表和消息表**

在 `sql/ry_20250522.sql` 末尾追加 `ai_chat_session` 和 `ai_chat_message` 表。主键使用 MySQL 自增，状态字段使用字符型以贴近 RuoYi 既有表风格。

```sql
drop table if exists ai_chat_session;
create table ai_chat_session (
  session_id     bigint(20)      not null auto_increment    comment '会话ID',
  user_id        bigint(20)      not null                   comment '用户ID',
  title          varchar(200)    default ''                 comment '会话标题',
  model          varchar(100)    default ''                 comment '模型名称',
  status         char(1)         default '0'                comment '状态（0正常 1删除）',
  create_by      varchar(64)     default ''                 comment '创建者',
  create_time    datetime                                   comment '创建时间',
  update_by      varchar(64)     default ''                 comment '更新者',
  update_time    datetime                                   comment '更新时间',
  remark         varchar(500)    default null               comment '备注',
  primary key (session_id),
  key idx_ai_chat_session_user (user_id, status, update_time)
) engine=innodb auto_increment=1 comment = 'AI聊天会话表';

drop table if exists ai_chat_message;
create table ai_chat_message (
  message_id     bigint(20)      not null auto_increment    comment '消息ID',
  session_id     bigint(20)      not null                   comment '会话ID',
  user_id        bigint(20)      not null                   comment '用户ID',
  role           varchar(20)     not null                   comment '角色（user assistant system）',
  content        longtext                                   comment '消息内容',
  status         char(1)         default '0'                comment '状态（0正常 1生成中 2失败）',
  error_message  varchar(1000)   default null               comment '失败原因',
  create_time    datetime                                   comment '创建时间',
  primary key (message_id),
  key idx_ai_chat_message_session (session_id, create_time),
  key idx_ai_chat_message_user (user_id, session_id)
) engine=innodb auto_increment=1 comment = 'AI聊天消息表';
```


- [ ] **步骤 4：实现发送消息核心流程**

Service 方法签名：

```java
void sendMessage(AiChatSendRequest request, Long userId, String username, SseEmitter emitter);
```

执行顺序：

1. 校验 `content` 非空。
2. 有 `sessionId` 时校验归属；无 `sessionId` 时创建会话。
3. 保存 `user` 消息。
4. 保存状态为 `1` 的 `assistant` 消息。
5. 发送 `session` 和 `message` 事件。
6. 读取最近 20 条上下文。
7. 调用模型适配器。
8. 累积增量并发送 `delta`。
9. 完成时更新助手消息状态为 `0`。
10. 异常时更新助手消息状态为 `2` 并发送 `error`。
